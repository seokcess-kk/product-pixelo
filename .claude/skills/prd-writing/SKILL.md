# PRD Writing

PRD(Product Requirements Document) 작성 가이드.

## PRD 목적

- 제품 방향성 명확화
- 팀 간 커뮤니케이션 기준
- 개발 범위 정의
- 성공 기준 설정

## 작성 가이드

### 좋은 PRD의 특징

1. **명확함**: 해석의 여지가 없음
2. **측정 가능**: 성공 여부 판단 가능
3. **범위 정의**: In/Out Scope 명확
4. **실현 가능**: 주어진 리소스로 달성 가능

### 기능 설명 팁

```markdown
# ❌ Bad
"로그인 기능"

# ✅ Good
"이메일/비밀번호로 로그인"
- 이메일 형식 검증
- 비밀번호 8자 이상
- 5회 실패 시 잠금
- 로그인 상태 7일 유지
```

### 완료 조건 팁

```markdown
# ❌ Bad
"로그인이 되면 완료"

# ✅ Good
"다음 조건 모두 충족 시 완료:
1. 올바른 credentials로 로그인 성공
2. 잘못된 credentials로 에러 메시지 표시
3. 로그인 후 대시보드로 리다이렉트
4. 새로고침해도 로그인 상태 유지"
```

## CEO 확인 포인트

PRD 작성 중 CEO 확인이 필요한 사항:

1. **핵심 기능 우선순위** — P0/P1 구분
2. **타겟 사용자 명확화** — 누구를 위한 건지
3. **성공 지표 정의** — 어떻게 성공을 판단할지
4. **MVP 범위 결정** — 뭘 넣고 뺄지
5. **브랜드 방향** — 제품의 톤앤매너
