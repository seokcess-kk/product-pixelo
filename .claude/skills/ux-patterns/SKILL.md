# UX Patterns

UX 디자인 패턴 및 사용성 원칙 가이드.

## Nielsen의 10가지 휴리스틱

| # | 원칙 | 적용 |
|---|------|------|
| 1 | **시스템 상태 가시성** | 로딩, 진행률, 저장 상태 표시 |
| 2 | **실제 세계와 일치** | 기술 용어 대신 일상 언어 |
| 3 | **사용자 제어와 자유** | Undo, 취소, 뒤로가기 |
| 4 | **일관성과 표준** | 동일 요소 = 동일 동작 |
| 5 | **에러 예방** | 확인 대화상자, 유효성 검사 |
| 6 | **기억보다 인식** | 최근 항목, 자동완성 |
| 7 | **유연성과 효율성** | 초보자/전문가 모두 만족 |
| 8 | **미니멀 디자인** | 불필요한 정보 제거 |
| 9 | **에러 복구 지원** | 명확한 에러 메시지 + 해결책 |
| 10 | **도움말 제공** | 툴팁, FAQ, 가이드 |

## 핵심 UX 법칙

### 3클릭 규칙
```
목표 달성까지 3클릭 이내

예외: 복잡한 작업은 3클릭 넘어도 OK
     단, 각 단계가 명확하고 진행감이 있어야 함
```

### 피츠의 법칙
```
중요한 버튼 = 크게 + 가까이

- 주요 CTA: 최소 44x44px
- 자주 쓰는 기능: 손/마우스 닿기 쉬운 위치
- 모바일: Thumb Zone 고려
```

### 힉의 법칙
```
선택지 ↑ = 결정 시간 ↑

- 한 화면에 선택지 7±2개 이하
- 많으면 그룹화/단계 분리
- 추천/기본값 제공
```

### 밀러의 법칙
```
단기 기억 용량: 7±2개

- 메뉴 항목 7개 이하
- 폼 필드 그룹화
- 긴 목록은 청킹(chunking)
```

### 제이콥의 법칙
```
사용자는 다른 서비스에서 익힌 패턴을 기대

- 익숙한 UI 패턴 사용
- 혁신보다 관습 (중요 기능일수록)
```

## 마찰(Friction) 최소화

### 체크리스트

**불필요한 단계**
- [ ] 이 단계 없이도 목표 달성 가능? → 제거
- [ ] 두 단계를 하나로 합칠 수 있나? → 합치기
- [ ] 나중에 해도 되나? → 뒤로 미루기

**인지 부하**
- [ ] 한 화면에 정보 너무 많나? → 분리/숨김
- [ ] 용어가 어렵나? → 쉬운 말로
- [ ] 뭘 해야 할지 명확한가? → CTA 강조

**입력 부담**
- [ ] 꼭 필요한 입력만? → 선택 필드는 선택으로
- [ ] 자동으로 채울 수 있나? → 자동완성, 기본값
- [ ] 타이핑 대신 선택? → 드롭다운, 버튼

## 플로우 다이어그램 표기법

```
[시작/끝]       : 둥근 사각형
<판단/분기>     : 다이아몬드
(액션/화면)     : 사각형
→              : 흐름 방향
```

### 플로우 예시

```
[진입]
   ↓
(로그인 화면)
   ↓
[이메일 입력] → [비밀번호 입력]
   ↓
(로그인 버튼 클릭)
   ↓
<인증 성공?>
   ↓ Yes          ↓ No
(대시보드)    (에러 메시지)
   ↓                ↓
[완료]        (로그인 화면)
```

## 와이어프레임 표기법

```
┌─────────┐  : 컨테이너
│         │
└─────────┘

[Button]    : 버튼
[___]       : 입력 필드
( ) Option  : 라디오 버튼
[x] Check   : 체크박스
─────────   : 구분선
☰           : 메뉴 아이콘
```

## 필수 상태 정의

모든 화면/컴포넌트에 정의:

| 상태 | 설명 | 필수 |
|------|------|------|
| Default | 기본 상태 | ✅ |
| Loading | 로딩 중 | ✅ |
| Empty | 데이터 없음 | ✅ |
| Error | 에러 발생 | ✅ |
| Success | 성공 | 상황별 |
| Disabled | 비활성화 | 상황별 |

## UX 트렌드 (2024-2025)

### 인터랙션
- Micro-interactions (작은 피드백 애니메이션)
- Meaningful transitions (의미 있는 화면 전환)
- Haptic feedback (모바일 진동)

### 네비게이션
- Bottom navigation (모바일)
- Floating action button
- Sticky elements

### 입력
- Progressive disclosure (점진적 공개)
- Inline validation (실시간 검증)
- Smart defaults (지능적 기본값)
- Passwordless auth

### 접근성
- Dark mode 필수
- Reduced motion 옵션
- Large text 지원

## 사용성 체크리스트

### 직관성
- [ ] 처음 사용자도 5초 내 다음 행동 알 수 있음
- [ ] 레이블/버튼 텍스트 명확
- [ ] 아이콘만으로 불충분하면 텍스트 병기

### 효율성
- [ ] 핵심 태스크 3클릭 이내
- [ ] 반복 작업에 단축 경로
- [ ] 불필요한 확인/단계 없음

### 학습 용이성
- [ ] 한 번 배우면 기억 가능
- [ ] 일관된 패턴
- [ ] 복잡한 기능은 온보딩/툴팁

### 에러 관리
- [ ] 실수하기 어렵게 설계
- [ ] 에러 메시지가 해결책 포함
- [ ] Undo로 복구 가능

### 만족감
- [ ] 완료 시 성취감 (피드백)
- [ ] 로딩/대기 지루하지 않음
- [ ] 시각적으로 현대적
