# Quality Checklist

코드 및 산출물 품질 검증을 위한 체크리스트 모음.

## 코드 품질 체크리스트

### 기본 품질

- [ ] TypeScript 컴파일 에러 없음 (`npm run type-check`)
- [ ] ESLint 경고/에러 없음 (`npm run lint`)
- [ ] Prettier 포맷팅 적용 (`npm run format`)
- [ ] 불필요한 console.log 제거
- [ ] 불필요한 주석 제거
- [ ] TODO/FIXME 주석 처리 완료

### 코드 구조

- [ ] 함수/컴포넌트가 단일 책임 원칙 준수
- [ ] 함수 길이 50줄 이하 권장
- [ ] 중복 코드 없음 (DRY)
- [ ] 적절한 추상화 레벨
- [ ] 명확한 네이밍 (의도 전달)

### 타입 안전성

- [ ] `any` 타입 미사용
- [ ] 적절한 타입 정의
- [ ] null/undefined 안전하게 처리
- [ ] 타입 가드 적절히 사용

## Frontend 체크리스트

### 기능

- [ ] 요구사항 100% 구현
- [ ] 모든 사용자 시나리오 동작
- [ ] 엣지 케이스 처리 (빈 데이터, 에러 등)
- [ ] 폼 유효성 검사 동작

### UI/UX

- [ ] 디자인 명세와 일치
- [ ] 로딩 상태 표시
- [ ] 에러 상태 표시
- [ ] 빈 상태 표시
- [ ] 성공 피드백 표시

### 반응형

- [ ] 모바일 (320px~) 정상 표시/동작
- [ ] 태블릿 (768px~) 정상 표시/동작
- [ ] 데스크톱 (1024px~) 정상 표시/동작
- [ ] 터치 타겟 44x44px 이상

### 접근성

- [ ] 키보드로 모든 기능 사용 가능
- [ ] Tab 순서 논리적
- [ ] Focus 상태 시각적 표시
- [ ] 시맨틱 HTML 사용
- [ ] 이미지 alt 텍스트
- [ ] 색상 대비 4.5:1 이상

### 성능

- [ ] 불필요한 리렌더링 없음
- [ ] 큰 번들 사이즈 피함
- [ ] 이미지 최적화 (next/image)
- [ ] 코드 스플리팅 적용

### 레퍼런스 대조

- [ ] 스타일 토큰 적용됨
- [ ] Primary Reference 느낌 유지
- [ ] shadcn 기본값 커스터마이징됨

## Backend 체크리스트

### API 설계

- [ ] RESTful 원칙 준수
- [ ] 일관된 응답 형식
- [ ] 적절한 HTTP 상태 코드
- [ ] API 스펙 문서 작성/업데이트

### 보안

- [ ] 입력 유효성 검증 (Zod 등)
- [ ] SQL Injection 방지
- [ ] 인증/인가 확인
- [ ] 민감 정보 응답에 미포함
- [ ] RLS 정책 설정 (Supabase)

### 에러 처리

- [ ] 모든 에러 케이스 핸들링
- [ ] 의미 있는 에러 메시지
- [ ] 에러 로깅

### 성능

- [ ] 인덱스 적절히 설정
- [ ] N+1 쿼리 없음
- [ ] 불필요한 데이터 조회 없음

## PR 리뷰 체크리스트

### 변경사항 확인

- [ ] PR 설명이 변경사항을 명확히 설명
- [ ] 관련 이슈/태스크 링크됨
- [ ] 변경 범위가 적절함 (너무 크지 않음)

### 코드 리뷰

- [ ] 로직이 올바름
- [ ] 엣지 케이스 고려됨
- [ ] 코딩 표준 준수
- [ ] 테스트 포함 (필요시)

### 테스트

- [ ] 로컬에서 동작 확인
- [ ] 기존 기능 영향 없음 (회귀)
- [ ] 자동화 테스트 통과

## 배포 전 체크리스트

### 코드

- [ ] 모든 테스트 통과
- [ ] TypeScript 에러 없음
- [ ] ESLint 경고 없음
- [ ] 빌드 성공

### 환경

- [ ] 환경 변수 설정 완료
- [ ] Secret 노출 없음
- [ ] Production 설정 확인

### 데이터베이스

- [ ] 마이그레이션 적용됨
- [ ] 롤백 계획 있음

### 최종 확인

- [ ] QA 테스트 완료
- [ ] Staging 환경 검증
- [ ] 승인 완료

## 심각도 분류

리뷰 피드백 심각도:

| 레벨 | 의미 | 예시 |
|------|------|------|
| 🔴 **Blocker** | 반드시 수정 필요 | 보안 취약점, 데이터 손실 위험 |
| 🟠 **Major** | 수정 권장 | 버그, 성능 문제 |
| 🟡 **Minor** | 개선 권장 | 코드 스타일, 가독성 |
| 🟢 **Suggestion** | 제안 | 더 나은 방법 제안 |
