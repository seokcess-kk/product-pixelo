# Workflow: Design First

디자인부터 시작하는 워크플로우. 프로토타입이나 디자인 검증이 먼저 필요한 경우.

---

## 개요

| 항목 | 내용 |
|------|------|
| **목적** | 디자인 검증 후 개발 |
| **사용 시점** | 새로운 UX 실험, 디자인 리뉴얼, 프로토타입 필요 |
| **핵심 원칙** | 디자인 확정 → 개발 (순차적) |

---

## 단계별 흐름

```
Phase 1: 디자인 탐색
    CEO → planner → ux-designer → ui-designer
    
Phase 2: CEO 확인 (반복 가능)
    ui-designer → planner → CEO
    
Phase 3: 개발
    planner → backend-dev + frontend-dev
```

---

## Phase 1: 디자인 탐색

### 1.1 요구사항 수집

**planner 작업:**
```markdown
@CEO 디자인 요구사항 확인

디자인 작업을 위해 확인이 필요합니다:

1. **목표**: 이 디자인으로 달성하려는 것?
2. **대상 화면**: 어떤 화면/기능?
3. **레퍼런스**: 참고하고 싶은 서비스/이미지?
4. **피하고 싶은 것**: 싫어하는 스타일?
5. **제약**: 기존 디자인과 일관성 필요? 완전히 새롭게?
```

### 1.2 UX 탐색

**ux-designer 작업:**
- 여러 플로우 옵션 탐색
- 와이어프레임 2-3개 옵션 제작
- 사용성 관점 장단점 분석

**산출물:**
```markdown
## UX 옵션 제안

### 옵션 A: {옵션명}
- 플로우: {설명}
- 장점: {장점}
- 단점: {단점}
- 와이어프레임: /design/wireframes/option-a/

### 옵션 B: {옵션명}
...

### 추천: 옵션 {X}
- 이유: {추천 이유}
```

### 1.3 UI 탐색

**ui-designer 작업:**
- 선택된 UX 기반 UI 옵션 탐색
- 스타일 방향 2-3개 제안
- 레퍼런스 기반 스타일 토큰 번역

**산출물:**
```markdown
## 디자인 방향 제안

### 옵션 A: {방향명}
- 컨셉: {한 줄}
- 레퍼런스: {참고 서비스}
- 컬러: {Primary Color}
- 느낌: {키워드}

### 옵션 B: {방향명}
...

### 추천: 옵션 {X}
```

## Phase 2: CEO 확인 (반복 가능)

### 2.1 디자인 제안

**planner 작업:**
```markdown
@CEO 디자인 방향 확인

## UX 옵션
{옵션 요약}
추천: {옵션}

## UI 옵션
{옵션 요약}
추천: {옵션}

## 문서
- UX: /design/wireframes/
- UI: /design/ui/

어떤 방향으로 진행할까요?
피드백 주시면 수정하겠습니다.
```

### 2.2 피드백 반영 (반복)

**CEO 피드백 → ui-designer/ux-designer 수정 → 다시 확인**

반복 조건:
- CEO가 만족할 때까지
- 또는 3회 반복 후 방향 결정

### 2.3 디자인 확정

**planner 작업:**
```markdown
## 디자인 확정

- 선택된 UX: {옵션}
- 선택된 UI: {옵션}
- CEO 승인: ✅

개발 단계로 진행합니다.
```

---

## Phase 3: 개발

### 3.1 스타일 토큰 적용

**frontend-dev 작업:**
- tailwind.config.js 업데이트
- 디자인 토큰 적용
- shadcn 컴포넌트 커스터마이징

### 3.2 구현

**backend-dev / frontend-dev 작업:**
- 확정된 디자인 기반 구현
- 레퍼런스 대조 검증

### 3.3 검수

**reviewer 작업:**
- 디자인 일치 확인
- 레퍼런스 느낌 확인

---

## 주의사항

1. **디자인 확정 전 개발 금지** — 디자인 변경 시 재작업 발생
2. **옵션 제시** — CEO에게 선택지 제공 (단일 안 X)
3. **빠른 피드백 사이클** — CEO 확인 빠르게 진행
4. **범위 관리** — 디자인 탐색이 무한 확장되지 않도록
