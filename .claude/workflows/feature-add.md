# Workflow: Feature Add

기존 프로젝트에 새로운 기능을 추가하는 워크플로우.

---

## 개요

| 항목 | 내용 |
|------|------|
| **목적** | 기존 프로젝트에 기능 추가 |
| **전제 조건** | 프로젝트, PRD, 디자인 시스템 존재 |
| **핵심 원칙** | 기존 패턴 유지, 회귀 방지 |

---

## 단계별 흐름

```
Step 1: 요구사항 분석
    CEO → planner → 기존 문서 확인
    
Step 2: 설계
    planner → ux-designer → ui-designer (필요시)
    
Step 3: 구현
    planner → backend-dev / frontend-dev
    
Step 4: 검수 & 완료
    reviewer → qa-engineer → planner
```

---

## Step 1: 요구사항 분석

### 1.1 기능 요청 접수

**planner 작업:**
```markdown
기능 추가 요청 접수: {기능명}

## 분석
- 기존 PRD: /docs/prd-{project}.md
- 관련 기능: {기존 관련 기능}
- 영향 범위: {예상 영향 범위}

## 확인 필요 사항
@CEO 다음 사항 확인 부탁드립니다:
1. {확인 사항 1}
2. {확인 사항 2}
```

### 1.2 기존 코드/문서 분석

**planner 작업:**
- PRD 확인
- 기존 코드 구조 파악
- 디자인 시스템 확인
- API 스펙 확인

### 1.3 태스크 분해

**planner 작업:**
```markdown
## TASK-{번호}: {기능명}

### 필요 작업
- [ ] UX 플로우 추가 (ux-designer) - 필요시
- [ ] UI 디자인 (ui-designer) - 필요시
- [ ] API 추가 (backend-dev) - 필요시
- [ ] 화면 구현 (frontend-dev)
- [ ] 테스트 (qa-engineer)

### 예상 영향
- 파일: {영향받는 파일}
- 기능: {영향받는 기능}
```

---

## Step 2: 설계 (필요시)

### 2.1 UX 설계 (필요시)

**ux-designer 작업:**
- 기존 플로우에 새 기능 통합
- 와이어프레임 추가/수정

**조건:** 새로운 화면이나 복잡한 플로우가 필요한 경우

### 2.2 UI 디자인 (필요시)

**ui-designer 작업:**
- 기존 디자인 시스템 활용
- 새 컴포넌트 필요시 추가
- UI 명세 작성

**조건:** 새로운 UI 요소가 필요한 경우

### 2.3 설계 검수

**reviewer 작업:**
- 기존 패턴과 일관성 확인
- UX 사용성 확인

---

## Step 3: 구현

### 3.1 Backend 구현 (필요시)

**backend-dev 작업:**
- API 스펙 업데이트
- API 구현
- DB 스키마 변경 (필요시)

**체크:**
- [ ] 기존 API와 일관성
- [ ] 기존 테이블 영향 최소화

### 3.2 Frontend 구현

**frontend-dev 작업:**
- 컴포넌트 구현
- 페이지 구현/수정
- API 연동

**체크:**
- [ ] 기존 컴포넌트 재사용
- [ ] 디자인 시스템 준수
- [ ] 스타일 토큰 사용

### 3.3 구현 검수

**reviewer 작업:**
- 코드 품질 검수
- 기존 코드 스타일 일관성
- 기능 동작 확인

---

## Step 4: 검수 & 완료

### 4.1 테스트

**qa-engineer 작업:**
- 새 기능 테스트
- **회귀 테스트** (기존 기능 영향 확인)
- 버그 리포트

### 4.2 버그 수정

**frontend-dev / backend-dev 작업:**
- 버그 수정
- 재검증

### 4.3 완료 보고

**planner 작업:**
```markdown
@CEO 기능 추가 완료

## 완료된 기능
- {기능 설명}

## 변경 사항
- {변경된 부분}

## 테스트 결과
- Pass: {N}건
- 회귀 테스트: 통과

## 확인 방법
{기능 확인 방법}
```

---

## 체크리스트

### 시작 전
- [ ] 기존 PRD 확인
- [ ] 기존 디자인 시스템 확인
- [ ] 영향 범위 파악

### 구현 중
- [ ] 기존 패턴/스타일 준수
- [ ] 새 패턴 도입 최소화
- [ ] 코드 일관성 유지

### 완료 전
- [ ] 새 기능 테스트 완료
- [ ] 회귀 테스트 완료
- [ ] reviewer 검수 통과

---

## 주의사항

1. **기존 패턴 우선** — 새 패턴 도입보다 기존 패턴 활용
2. **회귀 테스트 필수** — 기존 기능 깨짐 방지
3. **영향 범위 명시** — 변경되는 파일/기능 명확히
4. **문서 업데이트** — PRD, API 스펙 등 업데이트
