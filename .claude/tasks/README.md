# Tasks

planner가 관리하는 태스크 저장소.

## 폴더 구조

```
tasks/
├── _active/      # 진행 중인 태스크
├── _done/        # 완료된 태스크
└── _backlog/     # 대기 중인 태스크
```

## 태스크 파일 형식

파일명: `TASK-{번호}.md`

```markdown
# TASK-{번호}: {태스크 제목}

## 상태
- Status: Todo / In Progress / Review / Done
- 담당: {agent-name}
- 생성일: {날짜}
- 완료일: {날짜}

## 설명
{태스크 설명}

## 완료 조건
- [ ] {조건 1}
- [ ] {조건 2}

## 관련 문서
- PRD: {링크}
- 디자인: {링크}

## 히스토리
- {날짜}: 생성
- {날짜}: {담당}에게 할당
- {날짜}: 완료
```

## 태스크 흐름

```
_backlog/ → _active/ → _done/
(대기)      (진행)     (완료)
```

## 관리 원칙

1. **planner만 태스크 생성/이동**
2. **하나의 태스크 = 하나의 담당 Agent**
3. **완료 조건 명확히**
4. **히스토리 기록**
