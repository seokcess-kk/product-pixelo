# PIXELO - Main User Flow Overview

## Overview
- **서비스명**: 픽셀로 (PIXELO)
- **컨셉**: 일일 질문 응답 -> 성격 축 점수 계산 -> 픽셀 오브젝트 획득 -> 나만의 공간 완성
- **타겟**: MZ세대 (10대~2030)
- **플랫폼**: 모바일 웹앱

## 7개 측정축 시스템

### 측정축 구조

| # | 측정축 | 축 범위 | 설명 |
|---|--------|---------|------|
| 1 | 에너지 방향 | 내향 <-> 외향 | 에너지를 얻는 방식 |
| 2 | 생활 패턴 | 루틴 <-> 즉흥 | 일상의 계획성 |
| 3 | 감성 스타일 | 이성 <-> 감성 | 판단의 기준 |
| 4 | 미적 취향 | 미니멀 <-> 맥시멀 | 선호하는 스타일 |
| 5 | 사회적 성향 | 개인 <-> 협력 | 작업 선호 방식 |
| 6 | 도전 성향 | 안정 <-> 모험 | 새로운 것에 대한 태도 |
| 7 | 관계 방식 | 깊은관계 <-> 넓은관계 | 인간관계 패턴 |

### 점수 시스템

```
각 축: 0 ~ 100점 (50이 중간)
- 0~30: 왼쪽 성향 강함
- 31~49: 왼쪽 성향 약간
- 50: 중립
- 51~70: 오른쪽 성향 약간
- 71~100: 오른쪽 성향 강함
```

## 시즌 시스템

### 시즌 1: 나의 방
- **테마**: 개인 공간, 휴식처
- **오브젝트 카테고리**:
  - 가구 (침대, 책상, 의자 등)
  - 조명 (스탠드, 무드등 등)
  - 소품 (책, 식물, 액자 등)
  - 바닥/벽 (러그, 포스터 등)
- **완성 조건**: 필수 오브젝트 N개 배치

### 시즌 2: 나의 아지트 (예정)
- **테마**: 확장된 공간, 사회적 공간

## Navigation Structure

### Bottom Navigation (모바일)

```
+-------+-------+-------+-------+-------+
|       |       |       |       |       |
| 홈    | 질문  | 공간  | 친구  | 설정  |
|       |       |       |       |       |
+-------+-------+-------+-------+-------+
   *      (3)     *       (2)     *

* 새 알림/업데이트 있으면 뱃지 표시
(3) = 오늘 남은 질문 수
(2) = 새 친구 요청 수
```

## Key Interactions

### 1. 질문 응답 인터랙션
- 스와이프/탭으로 선택지 선택
- 선택 시 즉각적인 시각 피드백
- 진행률 표시 (3/5)
- 응답 후 다음 질문 자동 전환

### 2. 공간 배치 인터랙션
- 드래그 앤 드롭으로 오브젝트 배치
- 핀치 줌으로 공간 확대/축소
- 오브젝트 탭으로 회전/삭제 옵션
- 그리드 스냅 (선택적)

### 3. 공유 카드 인터랙션
- 템플릿 선택 (스와이프)
- 공유할 정보 선택 (토글)
- 미리보기 확인
- 원터치 SNS 공유

## States & Edge Cases

### 앱 상태별 처리

| 상태 | 처리 |
|------|------|
| 첫 방문 | 온보딩 -> 로그인 -> 초기 설정 -> 튜토리얼 |
| 재방문 (로그인) | 메인 홈 직행 |
| 재방문 (로그아웃) | 로그인 화면 |
| 오프라인 | 캐시된 공간 보기 가능, 질문/저장 불가 |
| 세션 만료 | 자동 재로그인 시도, 실패 시 로그인 화면 |

### 질문 관련 Edge Cases

| 상황 | 처리 |
|------|------|
| 질문 중 앱 종료 | 다음 접속 시 이어서 진행 |
| 질문 중 네트워크 끊김 | 로컬 저장 후 네트워크 복구 시 동기화 |
| 오늘 질문 모두 완료 | "내일 새 질문이 도착해요" 메시지 |
| 질문 건너뛰기 | 허용하지 않음 (응답 필수) |

## UX Principles Applied

### Nielsen Heuristics 적용

| 원칙 | PIXELO 적용 |
|------|-------------|
| 시스템 상태 가시성 | 질문 진행률, 오브젝트 획득 연출, 저장 상태 |
| 실제 세계와 일치 | "나의 방", "오브젝트", "친구" 등 친숙한 용어 |
| 사용자 제어와 자유 | 공간 편집 Undo, 응답 수정 (확정 전) |
| 일관성과 표준 | 동일한 버튼 스타일, 네비게이션 패턴 |
| 에러 예방 | 중요 액션 전 확인 (오브젝트 삭제 등) |
| 기억보다 인식 | 오브젝트 미리보기, 최근 획득 표시 |
| 유연성과 효율성 | 빠른 공유, 즐겨찾기 친구 |
| 미니멀 디자인 | 핵심 기능만 Bottom Nav에, 부가 기능 숨김 |
| 에러 복구 지원 | 실패 시 명확한 안내 + 재시도 버튼 |
| 도움말 제공 | 튜토리얼, 툴팁, FAQ |

### 3클릭 규칙 준수

| 목표 | 경로 | 클릭 수 |
|------|------|---------|
| 오늘의 질문 응답 | 홈 -> 질문 배너 | 2 |
| 공간 편집 | 홈 -> 공간 탭 -> 편집 버튼 | 3 |
| 친구 공간 방문 | 친구 탭 -> 친구 선택 | 2 |
| 공유 카드 생성 | 공간 -> 공유 버튼 | 2 |
