## 1. 개요

### 1.1 서비스 한 줄 설명

**픽셀로(PIXELO)** - 시덥잖은 질문으로 자기 발견, 픽셀 유니버스 구축

### 1.2 서비스 정보

| 항목 | 내용 |
|------|------|
| 서비스명 | 픽셀로 (PIXELO) |
| 슬로건 | "픽셀로 만나는 나의 세계" |
| 컨셉 | 일상적인 질문에 답하며 자신을 발견하고, 그 결과로 나만의 픽셀 유니버스를 구축 |
| 플랫폼 | 모바일 최적화 웹앱 (PWA) |

### 1.3 배경 및 목적

**문제 상황**:
- MZ세대는 자기 이해와 정체성 탐구에 높은 관심을 가지고 있음
- 기존 MBTI, 심리테스트는 일회성이며 지속적인 engagement가 부족
- 자기 발견 결과를 시각적으로 표현하고 공유할 수 있는 서비스 부재

**해결 방안**:
- 매일 가벼운 질문으로 부담 없이 자기 탐구
- 질문 응답이 픽셀 오브젝트로 시각화되어 성취감 제공
- 시즌제 운영으로 지속적인 engagement 유도
- 소셜 기능으로 친구와 비교하며 재미 요소 추가

**기대 효과**:
- 높은 리텐션율 (일일 질문으로 매일 방문 유도)
- 바이럴 확산 (공유 카드, 친구 비교 기능)
- 시즌제로 지속적인 콘텐츠 확장 가능

### 1.4 목표

1. **자기 발견**: 48개 질문을 통해 7개 측정축 기반의 자아 프로필 완성
2. **시각적 성취**: 픽셀 오브젝트로 나만의 공간 꾸미기
3. **소셜 경험**: 친구와 비교하고, 서로의 공간 방문하기
4. **지속적 참여**: 시즌제 운영으로 장기 engagement 확보

### 3.3 기능 상세

#### 3.3.1 일일 질문 응답

**설명**: 매일 3~5개의 가벼운 질문을 제공하고 사용자 응답을 수집

**동작 방식**:
1. 사용자가 홈 화면 진입
2. 오늘의 질문 카드 표시 (남은 질문 수 표시)
3. 질문 카드 탭하여 질문 상세 진입
4. 선택지 중 하나 선택 (2~4개 보기)
5. 선택 완료 시 점수 반영 + 오브젝트 획득 애니메이션
6. 다음 질문으로 이동 또는 완료 화면

**질문 구조**:
- 시즌당 24개, 총 48개 (MVP)
- 각 질문은 1~2개 측정축에 영향
- 각 선택지는 해당 축의 +/- 점수 부여

**제한 사항**:
- 하루 최대 5개 질문
- 응답한 질문 재응답 불가
- 자정 기준 리셋

**성공 조건**:
- 질문 정상 표시
- 응답 저장 및 점수 반영
- 일일 제한 동작
- 오브젝트 획득 정상 동작

#### 3.3.2 측정축 점수 계산

**7개 측정축**:

| 축 | 양극 | 설명 |
|---|------|------|
| 에너지 방향 | 내향 - 외향 | 에너지를 얻는 방식 |
| 생활 패턴 | 루틴 - 즉흥 | 일상 생활 스타일 |
| 감성 스타일 | 이성 - 감성 | 의사결정 방식 |
| 미적 취향 | 미니멀 - 맥시멀 | 선호하는 스타일 |
| 사회적 성향 | 개인 - 협력 | 업무/활동 선호도 |
| 도전 성향 | 안정 - 모험 | 변화에 대한 태도 |
| 관계 방식 | 깊은관계 - 넓은관계 | 인간관계 스타일 |

**점수 체계**:
- 각 축: 0 ~ 100점 (50이 중간)
- 초기값: 50점 (중립)
- 질문당 점수 변동: -10 ~ +10

**계산 알고리즘**:

```
새 점수 = 현재 점수 + (선택지 가중치 * 질문 영향도)

- 선택지 가중치: -2, -1, +1, +2 (선택지별)
- 질문 영향도: 5 (기본값)
- 범위 제한: min(0, max(100, 새 점수))
```

**예시**:
```
질문: "주말에 에너지를 충전하는 방법은?"
영향 축: 에너지 방향
선택지:
  A. 혼자 집에서 쉬기 (+10, 내향)
  B. 소수 친한 친구와 만남 (+5, 내향)
  C. 새로운 사람들과 모임 (-5, 외향)
  D. 파티나 대규모 행사 참여 (-10, 외향)

현재 점수: 50
A 선택 시: 50 + 10 = 60 (내향 방향)
```

#### 3.3.3 오브젝트 획득

**오브젝트 구조**:
- 총 오브젝트 수: 질문당 1개, 48개
- 시즌 1 (나의 방): 24개 오브젝트
- 시즌 2 (나의 아지트): 24개 오브젝트

**획득 방식**:
- 질문 응답 완료 시 해당 오브젝트 자동 획득
- 선택한 답변에 따라 오브젝트 외형/컬러 변형 가능

**오브젝트 타입**:

| 카테고리 | 예시 | 배치 공간 |
|----------|------|-----------|
| 가구 | 침대, 책상, 의자, 소파 | 나의 방 |
| 소품 | 조명, 화분, 액자, 러그 | 나의 방 |
| 가전 | TV, 스피커, 게임기 | 나의 방 |
| 야외 가구 | 벤치, 테이블, 해먹 | 나의 아지트 |
| 자연물 | 나무, 꽃, 돌 | 나의 아지트 |
| 조명 | 가로등, 랜턴, 캠프파이어 | 나의 아지트 |


#### 3.3.5 친구 비교 및 방문

**친구 추가 방식**:
- 닉네임 검색
- 고유 친구 코드 입력
- 카카오/네이버 친구 연동 (선택)

**비교 기능**:
- 레이더 차트에 나와 친구 점수 오버레이
- 각 축별 차이 수치 표시
- "우리는 00% 비슷해요" 종합 유사도

**방문 기능**:
- 친구 목록에서 친구 선택
- 친구의 픽셀 공간 읽기 전용 표시
- 좋아요/응원 남기기 (선택)

#### 3.3.6 공유 카드 생성

**공유 타입**:
1. **공간 캡처**: 내 픽셀 공간 이미지
2. **프로필 카드**: 측정축 결과 + 대표 오브젝트
3. **비교 카드**: 친구와 비교 결과

**카드 구성**:
- 픽셀로 로고
- 메인 이미지/차트
- 사용자 닉네임
- 시즌/진행률 정보
- 공유 링크 QR (선택)

**생성 방식**:
- 클라이언트: html2canvas 또는 Canvas API
- 서버: 이미지 생성 API (필요시)

## 4. 사용자 시나리오

### 시나리오 1: 신규 사용자 온보딩

**상황**: 지민이 친구가 공유한 픽셀로 카드를 보고 서비스에 접속

**흐름**:
1. 지민이 공유 링크 클릭하여 랜딩 페이지 진입
2. 서비스 소개 확인 후 "시작하기" 버튼 탭
3. 카카오 로그인 선택
4. 닉네임 설정 화면에서 닉네임 입력
5. 간단한 튜토리얼 (스와이프 3장)
6. 첫 번째 질문 화면으로 이동
7. 질문에 답변, 첫 오브젝트 획득
8. "나의 방"에 오브젝트 배치 안내

**성공 조건**:
- 온보딩 완료율 80% 이상
- 첫 질문 응답까지 이탈 최소화

### 시나리오 2: 일일 질문 응답

**상황**: 현우가 출근길 지하철에서 픽셀로 앱 실행

**흐름**:
1. 현우가 앱 실행, 홈 화면 진입
2. "오늘의 질문 3개가 기다리고 있어요" 카드 확인
3. 카드 탭하여 첫 번째 질문 진입
4. 질문 읽고 선택지 중 하나 선택
5. 오브젝트 획득 애니메이션 + 점수 변화 표시
6. 다음 질문으로 자동 이동
7. 3개 질문 완료 후 "오늘의 결과" 요약 화면
8. "내 공간 꾸미러 가기" 또는 "나중에" 선택

**성공 조건**:
- 일일 질문 완료율 70% 이상
- 평균 응답 시간 30초 이내

### 시나리오 3: 픽셀 공간 꾸미기

**상황**: 지민이 새로 획득한 오브젝트로 방 꾸미기

**흐름**:
1. 지민이 "나의 방" 탭 선택
2. 현재 방 상태 표시 + 인벤토리 버튼
3. 인벤토리 열어 새 오브젝트 확인 (New 뱃지)
4. 새 오브젝트 드래그하여 방에 배치
5. 위치 조정 및 회전
6. 기존 오브젝트도 재배치
7. "저장" 버튼으로 배치 완료

**성공 조건**:
- 오브젝트 배치 UX 직관적
- 저장 실패 없음

### 시나리오 4: 친구 비교 및 공유

**상황**: 지민이 친구 민지와 결과 비교 후 공유

**흐름**:
1. 지민이 "친구" 탭 진입
2. 민지 프로필 선택
3. "비교하기" 버튼 탭
4. 레이더 차트에 두 사람 결과 오버레이
5. "우리는 72% 비슷해요" 결과 확인
6. 각 축별 차이 확인
7. "비교 카드 만들기" 버튼 탭
8. 카드 이미지 생성, 카카오톡으로 공유

**성공 조건**:
- 비교 기능 사용률 측정
- 공유 전환율 30% 이상

### 6.3 데이터베이스 스키마 (주요 테이블)

```sql
-- 사용자
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT,
  nickname TEXT NOT NULL UNIQUE,
  avatar_url TEXT,
  friend_code TEXT UNIQUE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 측정축 점수
CREATE TABLE user_scores (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  axis_type TEXT NOT NULL, -- energy, pattern, emotion, aesthetic, social, challenge, relationship
  score INTEGER DEFAULT 50,
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, axis_type)
);

-- 질문
CREATE TABLE questions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  season_id INTEGER NOT NULL,
  order_num INTEGER NOT NULL,
  content TEXT NOT NULL,
  axes JSONB NOT NULL, -- 영향 축 배열
  object_id UUID REFERENCES objects(id)
);

-- 선택지
CREATE TABLE choices (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  question_id UUID REFERENCES questions(id),
  content TEXT NOT NULL,
  score_effects JSONB NOT NULL -- { "energy": 10, "pattern": -5 }
);

-- 응답 기록
CREATE TABLE responses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  question_id UUID REFERENCES questions(id),
  choice_id UUID REFERENCES choices(id),
  responded_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, question_id)
);

-- 오브젝트
CREATE TABLE objects (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  category TEXT NOT NULL,
  season_id INTEGER NOT NULL,
  space_type TEXT NOT NULL, -- room, hideout
  image_url TEXT NOT NULL,
  width INTEGER NOT NULL,
  height INTEGER NOT NULL
);

-- 사용자 오브젝트 인벤토리
CREATE TABLE user_objects (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  object_id UUID REFERENCES objects(id),
  variant TEXT, -- 선택지별 변형
  acquired_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, object_id)
);

-- 오브젝트 배치
CREATE TABLE placements (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  object_id UUID REFERENCES objects(id),
  space_type TEXT NOT NULL,
  position_x INTEGER NOT NULL,
  position_y INTEGER NOT NULL,
  rotation INTEGER DEFAULT 0,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 친구 관계
CREATE TABLE friendships (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  friend_id UUID REFERENCES users(id),
  status TEXT DEFAULT 'pending', -- pending, accepted
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, friend_id)
);
```

## 부록: 측정축 상세 정의

### A. 에너지 방향 (Energy Direction)

| 점수 | 성향 | 특징 |
|------|------|------|
| 0-30 | 강한 내향 | 혼자만의 시간에서 에너지 충전, 소수 깊은 관계 선호 |
| 31-45 | 내향 | 조용한 환경 선호, 혼자 또는 소규모 활동 |
| 46-55 | 중립 | 상황에 따라 유연하게 대처 |
| 56-70 | 외향 | 사람들과 어울리며 에너지 획득 |
| 71-100 | 강한 외향 | 활발한 사회적 활동, 새로운 만남 즐김 |

### B. 생활 패턴 (Life Pattern)

| 점수 | 성향 | 특징 |
|------|------|------|
| 0-30 | 강한 루틴 | 계획적, 규칙적 생활, 예측 가능성 중시 |
| 31-45 | 루틴 | 기본 루틴 유지, 적당한 계획 |
| 46-55 | 중립 | 상황에 따라 계획과 즉흥 혼합 |
| 56-70 | 즉흥 | 유연한 대처, 변화 수용 |
| 71-100 | 강한 즉흥 | 자유로운 생활, 즉흥적 결정 선호 |

### C. 감성 스타일 (Emotional Style)

| 점수 | 성향 | 특징 |
|------|------|------|
| 0-30 | 강한 이성 | 논리적 분석, 데이터 기반 결정 |
| 31-45 | 이성 | 이성적 판단 우선, 감정 고려 |
| 46-55 | 중립 | 이성과 감성의 균형 |
| 56-70 | 감성 | 감정과 직관 중시 |

| 점수 | 성향 | 특징 |
|------|------|------|

| 31-45 | 미니멀 | 깔끔한 스타일 선호 |
| 46-55 | 중립 | 상황에 따른 스타일 |
| 56-70 | 맥시멀 | 풍성한 장식, 다채로운 색상 |
| 71-100 | 강한 맥시멀 | 화려함, 개성 표현, 컬렉팅 |

| 점수 | 성향 | 특징 |
